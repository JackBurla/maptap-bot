Methodology

Source: open-index/hacker-news on HuggingFace — the full public HN dataset as monthly Parquet files (11 GB, 235 files). We used Burla to fan out to 235 parallel workers — one per monthly Parquet file — each downloading and processing its shard independently. Total: 47.8 million rows in 5.6 minutes (vs. ~27 min serially).

Zero engagement is defined strictly: score == 1 (submitter's auto-upvote only) and descendants == 0 (no comments) and not marked dead or deleted.

URL aliveness: a second Burla fan-out checked 47,198 unique URLs via HTTP GET (8s connect timeout, browser User-Agent, 236 workers at batch_size=200). 5.6 minutes. 51.1% dead.

The full pipeline is two remote_parallel_map calls: 

from burla import remote_parallel_map

# Phase 1 — 235 parquet files, one worker each, 47.8M rows in 5.6 min
results = remote_parallel_map(process_file, jobs, func_cpu=1, func_ram=4, grow=True)

# Phase 2 — 47,198 URLs, 236 workers at 200 each, 5.6 min (serial: ~13 hrs)
results = remote_parallel_map(check_batch_from_disk, jobs, func_cpu=1, func_ram=1, grow=True)

Meltdown score: rule-based signal from post text. No LLM. Tier 1 vocabulary (3 pts each): explicit give-up phrases ("i give up", "nobody cares", "final attempt", "spent N years"), year-of-effort confessions, explicit despair. Tier 2 (1.5 pts): supporting signals ("please", "repost", "ignored", "3 years", etc.). Plus: text length >600 words (+2), ALL CAPS ratio >5% (0–3), exclamation density >2% (0–2). Only posts with at least one Tier 1 hit are shown — T2-only posts produce too many false positives (version numbers like "v2", incidental use of "finally").
